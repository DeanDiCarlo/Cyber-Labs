## Definition of Done for a Daily Writeup
A day is complete when:
- [ ] `writeups/NN-topic-slug/` exists with `README.md`, `screenshots/`, and `artifacts/`
- [ ] `screenshots/` contains evidence or at least placeholder files if nothing was captured
- [ ] `artifacts/` contains sanitized outputs such as scans, logs, exports, or summaries
- [ ] The README includes all required sections in the correct order:
  - Summary
  - Objectives
  - Environment and Scope
  - Methodology (Setup, Steps Taken)
  - Observations
  - Analysis and Findings
  - Validation and Proof (Screenshots list, Artifacts list)
  - Final Results (Completed, Not completed or not attempted)
  - Lessons Learned
  - References
- [ ] The day has a primary win condition met, or the alternative win condition met
- [ ] Changes are committed and pushed with a clear message

## Daily Routine
1. Create the day folder and placeholder files.
2. Run the lab and collect evidence.
3. Save artifacts as text outputs and keep them sanitized.
4. Fill in the README using the template.
5. Commit and push.
